## Issues I Hit 
*Issue*: I got stuck at Step 5. I created the repository on GitHub, but I had no idea how to actually "open" it inside Cursor. I kept looking for a simple download button, but I realized I needed to "Clone" it to keep the connection alive.
*How I solved*: I went to the Cursor Chat (Ctrl+L) and asked. The AI walked me through the Ctrl + Shift + P shortcut and showed me how to paste the link into the "Git: Clone" command. It even helped me pick a folder on my desktop. This was the hardest part for me, but using the AI as a tutor helped me get past the "tech wall" and actually finish the task.

*Issue*: At the very end, I tried to commit my work, but the terminal kept telling me "nothing to commit, working tree clean." I was stuck because I could see my writing on the screen, but GitHub was still empty.
How I solved: I asked the AI what "working tree clean" means. It helped me realize two things: first, I hadn't actually saved the file (the little white dot in Cursor), and second, I had already committed the changes but hadn't "Pushed" them yet. Once I ran the git push command, everything finally uploaded. It taught me to pay attention to the small details in the IDE, like save icons and terminal messages.

*Issue*: After I wrote my README and saved it, I went to GitHub and nothing was there. I thought I had failed the task or that my computer wasn't connected properly.
*How I solved*: I asked the Cursor AI, "Why can't I see my changes on GitHub?" It explained that in professional tools, "Saving" is not the same as "Pushing." I learned that I have to Commit (save a snapshot) and then Push (upload to the internet). I found the Source Control button in the sidebar, clicked 'Sync,' and finally saw my work appear online.

## My Takeaway: 
This was my first time using Cursor and GitHub together. While I hit a few bumps, especially with "cloning" the project and using the terminal, I'm proud that I used the AI tools available to me to find the answers.
